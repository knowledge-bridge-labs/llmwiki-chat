# Code of Conduct

LLMWiki Chat is intended to be a practical, respectful open-source project.
Contributors and maintainers are expected to keep discussions focused on the
software, the documentation, and the people using it.

## Scope

This code of conduct applies in project issues, pull requests, reviews,
documentation discussions, community chat, and maintainer interactions connected
to this repository.

## Expected Behavior

- Be respectful and constructive in issues, pull requests, reviews, and chat.
- Explain tradeoffs with evidence when you disagree.
- Keep feedback specific to the work and avoid personal attacks.
- Help new contributors understand project conventions without gatekeeping.
- Respect privacy and do not publish credentials, private exports, private
  endpoint details, or sensitive data.
- Keep review comments actionable and tied to project behavior, compatibility,
  documentation, tests, security, or maintainability.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information without permission.
- Posting credentials, tokens, private endpoint details, or sensitive exports.
- Repeatedly derailing technical discussions after maintainers ask to refocus.
- Submitting intentionally harmful code, documentation, tests, or artifacts.
- Misrepresenting validation results or security impact in public collaboration
  spaces.

## Reporting and Enforcement

Report conduct concerns through the private maintainer or organization contact
published for this repository. If no private contact is listed yet, open a
public issue only to request a private conduct contact path, then stop. Do not
include incident details, private data, credentials, or screenshots in that
public request. Do not open a public pull request or discussion for conduct
reports.

Include the relevant link, participants, and context needed to evaluate the
concern only after a private path is available. For suspected security
vulnerabilities, use the process in `SECURITY.md` instead.

Maintainers may edit, hide, or remove comments, close issues, reject pull
requests, request changes, or temporarily restrict participation when behavior
harms the project or its contributors.

Maintainers should apply enforcement consistently, protect reporters' privacy as
much as practical, and avoid public disclosure of sensitive details while a
conduct concern is being handled.
